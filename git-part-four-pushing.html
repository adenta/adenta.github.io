Pushing

This section of the tutorial assumes your git repo looks like this.

We need to push our files to our newly created repo. We last left off at this page:

Notice this snippet the page asks us to run if we have an existing repo:

git remote add origin git@github.iu.edu:NAME/git-training.git
git push -u origin master

Lets walk through what we are doing. First, ‘git remote add origin …’ Is adding a new remote repository location to your git repo on your computer. the remote that we are adding has the name of origin, and is located at git@github.iu.edu:NAME/git-training.git (where NAME is a stand-in for your name). We could add a bunch of different remote servers in this manner, but we only need one.

Now we tell git to push the repo from our local computer to the GitHub repository with the git push -u origin master command.

And thats it! If you did not recieve error messages, you should see your repository on your Github Account!
