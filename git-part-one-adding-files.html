Adding and Staging Files.

Cloning a repo

This section of the tutorial will cover how to add files to a repository. This section will also cover what it means to ‘stage’ and ‘commit’ files, and how both of these actions are different from a ‘push’.

For the remainder of the tutorial, we will focus on Shoes for Snakes. This is a fictitious non-profit we made up, who we will be helping get their website ready for production.

First, we will need to ‘clone’ our git repository. Think of a git repository as a bunch of files. We need to get these files onto whatever computer we are working on. To do this, we use the git clone command. The git clone command works by typing:

Here, we type ‘git clone’ to specify we want to clone a repository somewhere else. we follow this command with the URL of the repository. After you run this command through your terminal, you will have the repository on your desktop.

Adding a file

Now that we have our repository on our computer, we can look at the files, and make changes to it. If we open the ‘index.html’ in a web browser, we see the starts of a public website. If we look at the index.html in a code editor, we see that the style tag contains our style information.

If we want more than one page reading the same set of styles, we will need to move it to a separate file. To do this, we will create another file, called ‘style.css’. Create the style.css file in the same ‘git-tutorial-repo’ folder.

Staging the file

Now that we added the file, we have to stage it. When you add a file to a git repo, we have to tell git to track the file. This makes more sense when working in larger projects. To add our new ‘style.css’ file, we run the following in our git-tutorial-repo folder:

git add style.css

This does what you would imagine: It adds the style.css file to the git repository. Now whenever we change the file, git will keep track of the change. Git will only track the change when we commit the file to the repository. We will cover Commits in the next section.

Advanced add command

Sometimes, we want to add a bunch of files. To do that we:

git add .

This recursively adds all untracked files in the current git repository. If I made changes to the ‘index.html’ and the ‘style.css’ files, I could add them both by typing ‘git add .’.
