Committing and Removing Files

This section of the tutorial will talk about the ‘commit’ step of the git work flow. We will also look at what happens when you remove a file from a git repo, and how to tell git about that changed file.

To watch this tutorial, please visit asciinema.org
This terminal recording can be found at asciinema.org.

Committing a repo

Now that we made changes to our files, we want to save a ‘snapshot’ of the current project. This snapshot is called a commit. Each commit also has an attached message, that gives some information about the commit. To commit your files, we will type:

git commit -m "put styles in a separate file."

Here, we are telling git to remember what the repository looked like at this point in time, and attaching a message of “put styles in a separate file.” This will help us remember in the future where the first commit was that added the style.css file.
